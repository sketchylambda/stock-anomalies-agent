## ✨ Core Features
* **Live Market Triage:** Dynamically builds the S&P 500 index and fetches 1-month historical pricing for all 500 assets in seconds using `asyncio` and `yfinance`.
* **Statistical Filtration:** Instantly filters out market noise, isolating stocks experiencing mathematical outlier events (2-Sigma and 3-Sigma divergences).
* **Multi-Layered AI Pipeline:** Features dedicated agents for market analysis, customer support, and database querying, all protected by a strict LLM-as-a-judge security guardrail.
* **Smart Database Caching:** Uses Google BigQuery to cache AI analyses for the duration of the trading day, dramatically reducing API token costs.
* **Admin Export Pipeline:** Secure, authenticated backend route to query user-reported anomalies and generate downloadable CSV reports.

## 🤖 The AI Agent Ecosystem

AlphaPulse relies on a specialized, multi-agent architecture within its Python backend (`agents/analyst.py`) to process data safely and efficiently.

### 1. The Market Analyst Agent 📊
* **Function:** Ingests the statistical payload (Ticker, Current Price, Z-Score) and generates a concise, 2-sentence executive summary explaining whether the statistical anomaly is likely market noise or a fundamental shift.

### 2. The Security Guardrail (LLM-as-a-Judge) 🛡️
* **Function:** An enterprise-grade safety mechanism operating at `temperature=0.0`. It intercepts every analysis generated by the Market Analyst and evaluates it strictly to detect direct financial advice. If restricted advice is detected, the output is blocked and replaced with a safety warning.

### 3. The Customer Support Agent 💬
* **Function:** An interactive client liaison. It helps users navigate the platform and defines complex financial metrics (like P/E, ROE, and Z-Scores) dynamically. It operates under strict system instructions to never provide stock recommendations.

### 4. The DAH (Data Analyst Helper) Agent 🧠
* **Function:** Built on the Google Agent Development Kit (`google.adk`), this advanced agent acts as the internal quantitative researcher. It is equipped with a `BigQueryToolset` (with `WriteMode.BLOCKED` for safety) and `InMemorySessionService` to answer complex questions about the BigQuery database state, fetch data, and write raw SQL for engineers.

## 📡 System Architecture

![AlphaPulse Functional Flow](assets/architecture.png)

AlphaPulse is built on a high-concurrency, serverless framework designed to bridge the gap between raw statistical processing and agentic reasoning. The architecture follows a Decoupled Multi-Agent Pipeline, where specialized AI roles are strictly separated by security guardrails and data persistence layers.

Unlike traditional chatbots, AlphaPulse utilizes an Agent-to-App integration pattern. The backend doesn't just "call an LLM"; it orchestrates a lifecycle where agents interact with live system state:

1. **Reactive Ingestion**: The FastAPI engine performs a non-blocking asyncio scan of the S&P 500, calculating Z-scores locally to minimize latency.

2. **Stateful Handoff**: Only "Critical" events (>= 2.0 Sigma) are handed off to the AI Layer.

3. **Click-to-Commit Logic**: Data is only persisted to BigQuery when a user interacts with a card. This ensures the database remains a curated "High-Signal" archive rather than a dump of raw market noise.

## 🧮 The Math: 3-Sigma Divergence

AlphaPulse relies on strict standard deviation modeling to identify assets acting outside their expected historical behavior, ignoring lagging technical indicators like RSI or MACD.

![3-Sigma Divergence Concept](assets/3-sigma-divergence.png)

The system calculates the Z-Score for every asset using the standard formula:
$$Z = \frac{X - \mu}{\sigma}$$

> **🟨 Yellow Alerts (Elevated):** Z-Score between 2.0 and 2.9 (roughly 95% of normal price action falls inside this band).
> 
> **🟪 Pink Alerts (Extreme):** Z-Score >= 3.0. These are rare, multi-standard-deviation events where the price has completely detached from its 30-day mean.

## 💻 Getting Started

<details>
<summary><b>Click to expand local setup instructions</b></summary>

### 1. Installation
Clone the repository and install the strict dependencies:
```bash
git clone [https://github.com/yourusername/alphapulse.git](https://github.com/yourusername/alphapulse.git)
cd alphapulse
pip install -r requirements.txt
```
### 2. Environment Variables
Create a `.env` file in the root directory:
```bash
GEMINI_API_KEY="your_google_ai_key"
GCP_PROJECT_ID="your_google_cloud_project_id"
GOOGLE_APPLICATION_CREDENTIALS="gcp-credentials.json" # Only needed for local dev
```
### 3. Run the Server
Start the Uvicorn ASGI server:
```bash
uvicorn app.main:app --reload
```
Navigate to `http://localhost:8000` in your browser.
